Webhook 模块允许 Crowdin Apps 监听特定事件，并在这些事件发生时触发相应操作。 该模块适用于自动化工作流（例如，与 [工作流步骤类型模块](/zh/developer/crowdin-apps-module-workflow-step-type/) 配合使用）、同步外部系统或记录项目活动。

## 结构

```json title="manifest.json"
{
  "modules": {
    "webhook": [
      {
        "key": "webhook-key-module-key",
        "url": "/webhooks",
        "events": [
          "project.translated",
          "project.approved"
        ],
        "environments": [
          "crowdin",
          "crowdin-enterprise"
        ]
      }
    ]
  }
}
```

## 应用与 Crowdin 之间的通信

所有 webhook 均通过 HTTP POST 请求以 JSON 格式的负载触发。 请求可包含单个事件或多个事件（始终以批量模式发送）。

<ReadMore>
  详细了解 [Webhook 负载示例](/zh/developer/webhooks/#webhook-负载示例)。
</ReadMore>

### 向应用发出请求

当事件发生时，Crowdin 会向您的 webhook 端点发送一个 HTTP POST 请求，其中包含用于身份验证和安全的必要标头。 以下是此类请求的示例。

**HTTP 请求：**

```
POST /webhooks HTTP/1.1
User-Agent: Crowdin-Webhook
Content-Type: application/json
X-Crowdin-Id: <organization-id>
X-Crowdin-Domain: <organization-domain>
X-Crowdin-Signature: <signature>
X-Module-Key: <your-module-key>
```

**请求正文：**

```json
{
  "events": [
    {
      "event": "project.translated",
      "project": {
        "id": "777",
        // ...
      },
      "targetLanguage": {
        "id": "uk",
        // ...
      }
    },
    {
      "event": "project.approved",
      "project": {
        "id": "777",
        // ...
      },
      "targetLanguage": {
        "id": "uk",
        // ...
      }
    }
  ]
}
```

在应用安装过程中，会向您的应用提供一个名为 [`app_secret`](/zh/developer/crowdin-apps-installation/#已安装事件通信流程) 的唯一密钥。 该密钥用于通过 HMAC 算法结合 SHA-256 生成 webhook 签名。

### 应用的预期响应

Crowdin 要求您的应用返回成功的 HTTP 状态码（即任意 2XX 状态）以确认收到 webhook 负载。
