当您准备好分享您的应用时，可以将其提交至 [Crowdin 商店](https://store.crowdin.com)。 发布后，其他 Crowdin 用户即可安装和使用您的应用。 但在发布应用之前，您需要在 Crowdin 商店获取一个作者账户。 为此，请[联系我们的客户成功服务](https://crowdin.com/contacts)并按照提供的说明操作。

## 创建作者页面

每位在 Crowdin 商店发布应用的开发者都将拥有一个个人作者页面。

以下是作者页面示例：[Crowdin](https://store.crowdin.com/author/5)。

获取作者账户后，您可以继续创建作者页面。 为此，请导航至[作者管理页面](https://developer.app.crowdin.net/admin/content/author)并点击**创建条目**。 您需要为您的作者页面提供一些信息。 我们建议在开始发布流程之前完成此操作。

在下表中，您可以查看作者页面需要填写的字段。

| 字段     | 描述                                                             | 是否必填 |
| ------ | -------------------------------------------------------------- | ---- |
| 状态     | 页面的发布状态。 在填写所有必填信息并准备好发布之前，请将其保留为**草稿**状态。                     | 是    |
| 标志     | 作者的徽标。 我们建议使用透明背景、最低分辨率为 256×256 像素的 .png 图片。                  | 是    |
| 描述     | 添加不超过 200 个字符的简短描述。                                            | 是    |
| 支持 URL | 可以是支持页面的链接。                                                    | 是    |
| 电子邮件   | 作者的电子邮件。                                                       | 是    |
| 名称     | 作者的显示名称。                                                       | 是    |
| URL    | 可以是企业网站的链接。                                                    | 否    |
| 验证     | 定义作者是否已通过 Crowdin 团队的验证。 在作者页面添加**已验证作者**徽章。 只能由 Crowdin 团队设置。 | 否    |

### 应用类型

应用类型列表：

| 类型    | 描述                                                                                                                |
| ----- | ----------------------------------------------------------------------------------------------------------------- |
| 应用程序  | 常规 Crowdin 应用程序（例如，项目集成、编辑器面板、资源或报告应用）。 应具有清单。                                                                    |
| 指南    | 描述某种集成或功能的条目。 通常包含外部链接。                                                                                           |
| QA 检查 | 自定义 QA 检查。 仅在 Crowdin Enterprise 中可用。 要查看示例，请访问 Crowdin 商店上的 [QA 检查部分](https://store.crowdin.com/types/qa-check)。 |
| 文件    | 自定义文件格式。 应具有清单。                                                                                                   |
| 系统    | 原生 Crowdin 集成或文件扩展名。                                                                                              |

### 应用描述

**内容**字段对于帮助用户了解您的应用价值至关重要。 以下是一些创作引人注目的应用描述以将访客转化为用户的技巧。

#### 构建内容结构

结构清晰的描述有助于用户快速了解您的应用功能：

1. **以价值为先** - 从您的应用提供的主要优势开始
2. **说明工作原理** - 描述关键功能和工作流程
3. **包含使用场景** - 帮助用户设想如何使用您的应用
4. **添加视觉内容** - 截图使您的应用更加直观

#### 使用告示块

您可以在应用描述中使用告示块来突出显示重要信息。 支持以下告示块类型：

```md
:::note
使用注释提供额外的上下文或阐明有关应用功能的重要细节。
:::

:::tip[专业提示]
提示非常适合分享最佳实践或建议使用应用的最佳方式。
:::

:::info
使用信息块突出显示关键功能或兼容性信息。
:::

:::warning
警告帮助用户了解安装前应注意的限制、先决条件或潜在问题。
:::

:::danger
谨慎使用危险提示来突出显示关键信息，例如无法撤销的操作或潜在的数据丢失场景。
:::
```

### 应用定价

定价字段以 JSON 格式描述应用的定价模型。 它仅用于在应用页面上渲染定价信息。 您还应在应用中实现相应的行为。

<ReadMore>
  阅读更多关于 [Crowdin 应用变现](/zh/developer/crowdin-apps-monetization/)的内容。
</ReadMore>

如果您的应用免费，只需在定价字段中留下空数组 `[]`。 如果您的应用为付费应用，请点击**填充模板值**并调整定价描述。

示例：

```json
[
  {
    "plan_type": "recurring",
    "price": {
      "monthly": "80",
      "yearly": "800",
      "currency": "USD"
    },
    "trial": true
  }
]
```

其中：

| 字段 <div class="w-28" /> | 描述                                                                           |
| ----------------------- | ---------------------------------------------------------------------------- |
| `plan_type`             | 应用计划的类型。 可选值：`recurring` 和 `free`。                                           |
| `price`                 | 应用的价格。 应包含一个具有 `monthly` 和/或 `yearly` 值的对象。 可用的 `currency` 值为 "USD" 和 "EUR"。 |
| `trial`                 | 定义应用程序是否有试用期。 默认情况下，crowdin.com 为 14 天，Crowdin Enterprise 为 30 天。            |

## 应用审核流程

应用程序在 Crowdin 将其列入 Crowdin 商店之前，需要经过审核流程。 审核流程包括审查 Crowdin 商店各自页面上的作者和应用描述。

### 作者验证

您可以为您的组织申请作者验证。 **已验证作者**徽章是显示在 Crowdin 商店中作者账户名称旁边的标记，表示 Crowdin 已确认该作者账户是其所代表的公司或个人的真实存在。

要申请作者验证，请[联系我们的客户成功服务](https://crowdin.com/contacts)，我们将就后续步骤达成一致。
