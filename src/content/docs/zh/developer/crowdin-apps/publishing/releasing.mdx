一旦您的 Crowdin 应用程序的开发阶段完成，您应该发布它以供用户和客户访问。

发布过程包括以下阶段：

* 设置和部署生产环境
* 将您的 Crowdin 应用程序发布到 [Crowdin 商店](https://store.crowdin.com/)

### 配置生产环境

在准备发布 Crowdin 应用程序时，请确保它同时具有生产和开发环境。 拥有独立环境有很多好处，但主要有以下几点：

* **稳定性** &ndash; 拥有多个环境可以减少或消除停机时间，从而避免应用程序流失客户。 它还将防止生产数据丢失或删除的事件。
* **安全性** &ndash; 环境隔离将限制无权访问客户数据的人员访问这些数据。

Crowdin 应用程序必须在环境之间使用单独的 OAuth 客户端、数据库和其他凭证。

<Aside>
  如果您将 Crowdin 应用程序设计为同时用于 Crowdin 和 Crowdin 企业版，请[联系支持团队](https://crowdin.com/contacts)，我们将使您的 OAuth 客户端适用于 Crowdin 和 Crowdin 企业版，这样您就无需为每个平台单独创建 OAuth 客户端。
</Aside>

### 部署 Crowdin 应用程序

当 Crowdin 应用程序准备好使用时，您需要在线托管它以供用户访问。 您可以根据自己的喜好选择任何首选平台。 唯一的要求是平台必须可靠。

您可以在下面看到可用于部署的平台列表：

* **Vercel** - 提供开发者体验和基础架构，用于构建、扩展和保护更快、更个性化的 Web 应用。 了解更多关于[入门](https://vercel.com/docs)和[部署](https://vercel.com/docs/platform/deployments)的信息。

* **Heroku** - 基于容器的云平台即服务（PaaS）。 了解更多关于 [Heroku 入门指南](https://devcenter.heroku.com/start)和[部署](https://devcenter.heroku.com/categories/deployment)的信息。

* **DigitalOcean** - 一家云计算供应商，为软件开发者提供基础架构即服务（IaaS）平台。 了解更多关于[入门](https://docs.digitalocean.com/products/app-platform/how-to/#getting-started)和[开发指南](https://docs.digitalocean.com/products/app-platform/languages-frameworks/)的信息。

* **Amazon Web Services** - 一个提供可扩展且经济高效的云计算解决方案的在线平台。

* **Google Cloud** - 一个提供可扩展云计算服务、数据分析以及用于构建和管理应用程序工具的平台。

#### 在 Crowdin 服务器上托管您的 Crowdin 应用程序

所有创建自己 Crowdin 应用程序的开发者都有机会将其应用程序托管在 Crowdin 服务器上。 要使用此选项，应用程序开发者需要将应用程序的源代码提交给 Crowdin 进行审核和部署。 我们的团队将审核源代码，一旦获得批准，该应用程序将由 Crowdin 托管。

每项应用程序托管安排都附有专门针对该应用程序定制的个性化协议。 此外，由 Crowdin 托管的应用程序将获得特殊的 <Badge text="Verified by Crowdin" variant="success" /> 徽章，从而增强潜在用户的信心，提高安装和使用的可能性。

[联系支持团队](https://crowdin.com/contacts)，我们将很乐意为您提供更多相关信息。

### 分发 Crowdin 应用程序

一旦您准备好与用户分享您的 Crowdin 应用程序，请[联系支持团队](https://crowdin.com/contacts)，为您提供 [Crowdin 商店](https://store.crowdin.com)上的开发者账户。 开发者账户让您可以在 Crowdin 商店中发布 Crowdin 应用程序，并查看不同的统计数据，例如安装数量、应用程序评级等。
