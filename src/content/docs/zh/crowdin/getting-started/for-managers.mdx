> Crowdin 是一个本地化管理平台，可帮助公司、团队和个人灵活翻译移动、桌面和 Web 应用程序、网站、游戏、帮助文档等产品。

在这篇文章中，我们将带您了解 Crowdin 中项目管理员的典型本地化工作流程。 我们将介绍创建项目、上传内容、邀请项目成员、自动翻译项目、生成报告以及下载译文的相关内容。

## 创建项目

创建一个项目，以便能够上传您的内容进行本地化。 创建项目时，您可以选择偏好的*项目可见性*、*项目类型*，以及源语言和目标语言。

<Image src={createProject} alt="创建一个项目" />

<ReadMore>
  详细了解[创建项目](/zh/creating-project/)。
</ReadMore>

创建项目后，您可以添加要翻译的内容。 每个项目都有自己的源内容、资源、目标语言、集成、成员等。

<Aside>
  一旦您创建了项目，就开始了 **14 天**免费试用期。
</Aside>

## 上传内容

项目创建完成后，上传可本地化的文件和资源，例如屏幕截图、翻译记忆和术语库。

### 上传源文件以供本地化

您可以使用最适合您的方式上传可本地化的文件：

* 通过网络界面（UI）手动进行。
* 将 Crowdin 与您已使用的工具集成：
  * 代码仓库 *(GitHub、GitLab 等)*
  * 营销工具 *(Contentful、Webflow、WPML、Hubspot、Marketo 等)*
  * 帮助中心 *(Help Scout、Zendesk、Intercom 等)*
  * 开发工具 *(VS Code、Android Studio、Google Play 等)*
  * 电子邮件自动化 *(MailChimp、SendPulse、ActiveCampaign、Mandrill、Moosend 等)*
  * 设计工具 *(Figma、Sketch、Adobe XD)*。
* 使用 API 或命令行客户端。

### 上传现有译文

如果您已经在 Crowdin 之外完成了一些翻译，您可以将它们上传到您的 Crowdin 项目，以将所有内容保留在一处并继续进行翻译工作。

<ReadMore>
  详细了解[上传现有译文](/zh/uploading-translations/)。
</ReadMore>

### 上传本地化资源

在大多数情况下，您可能拥有本地化资源，例如[翻译记忆](/zh/translation-memory/)、[术语库](/zh/glossary/)和[风格指南](/zh/style-guide/)。 将它们上传到您的 Crowdin 项目是个好主意，因为这些本地化资源可以在翻译过程中帮助您的翻译员并确保较高的翻译质量。

### 上传截图

屏幕截图是提供额外语境的另一种好方法，以便翻译员可以更好地理解他们应该如何翻译源字符串。 要上传屏幕截图，请打开您的项目并前往**屏幕截图**选项卡。 在屏幕截图上标记源字符串。 这样，带有标记字符串的屏幕截图将出现在编辑器的语境和译文部分中每个字符串的下方。

## 自动翻译您的项目

为了节省时间，您可以使用 [AI](/zh/crowdin-ai/)、[翻译记忆](/zh/translation-memory/)（TM）或[机器翻译](/zh/machine-translation/)（MT）对您的 Crowdin 项目进行自动翻译。 自动翻译内容意味着在翻译员开始工作之前，部分字符串将已被翻译。

您可以手动应用自动翻译，也可以设置自动自动翻译，这样每次添加新内容时，都会根据您的设置自动翻译。

<ReadMore>
  详细了解[自动翻译](/zh/auto-translation/)。
</ReadMore>

## 邀请成员

决定由谁来翻译和管理您的内容。 您可以与集成了 Crowdin 的专业翻译机构（即供应商）合作，也可以邀请自己的团队。

如果您决定与机构合作，请前往**商店 > 供应商**，选择符合您需求的翻译供应商。

<ReadMore>
  详细了解[订购专业译文](/zh/translation-strategies/#订购专业译文)。
</ReadMore>

如果您决定邀请自己的团队，请打开您的项目并前往**成员**选项卡，通过 Crowdin 用户名、电子邮件或邀请链接添加成员。 根据每个人在项目中的职责，您可以分配以下角色：管理员、开发者/翻译请求者、语言协调员、校对员和翻译员。

<Image src={inviteDialog} alt="邀请项目成员" class="width-xl crowdin-modal m-auto" />

<ReadMore>
  详细了解[邀请成员](/zh/inviting-people/)。
</ReadMore>

加入项目后，团队成员可以根据其角色开始在[编辑器](/zh/online-editor/)中工作或访问其他项目区域。

## 管理语言请求

项目成员可以通过点击**请求新语言**，直接从项目的**仪表板**选项卡请求新的目标语言。 作为项目管理员，您将以通知的形式收到这些请求，然后可以进行审核并在需要时添加新语言。

为了有效管理这些请求，您首先需要确保您的通知已正确配置。

### 启用请求通知

要在用户请求语言时收到提醒，您必须在**账户设置**中启用相应的通知。

<Steps>
  1. 前往**账户设置 > 通知**。
  2. 为您偏好的通知渠道（例如，应用内、电子邮件、Slack）选择**语言请求**。
</Steps>

<Aside type="tip">
  为了及时审核，建议为您经常查看的渠道（例如电子邮件或 Slack）启用通知。
</Aside>

### 审核语言请求

一旦用户请求新语言，您将收到通知。

<Steps>
  1. 点击右上角的铃铛图标 <Icon name="mdi:bell-outline" class="inline-icon" /> 以在下拉列表中查看新通知。
  2. 要查看所有通知的完整列表，请点击**查看全部**。
  3. 在**通知**页面，找到该请求。 通知将显示项目、请求的语言、发送请求的用户以及他们提供的任何备注。
  4. 点击**语言设置**，直接前往项目的**设置 > 语言**页面，将请求的语言添加到项目中。
</Steps>

## 生成报告

Crowdin 提供项目级报告，帮助您跟踪进度、评估翻译准确性、估算和统计费用，以及衡量贡献者活动。 要访问报告，请打开您的项目并前往**报告**选项卡。

可用报告包括：

* 概述
* 费用估计
* 翻译费用
* 自动翻译准确性
* 翻译员准确性
* 最佳成员
* 归档

## 下载翻译

您可以使用 Web 界面（UI）手动下载译文，也可以通过集成（例如 GitHub 或其他已连接的工具）、Crowdin API、命令行客户端（CLI）或 CDN 分发设置自动交付。

译文将根据项目**设置**选项卡中配置的导出设置进行下载。

  详细了解[下载译文](/zh/downloading-translations/)。
