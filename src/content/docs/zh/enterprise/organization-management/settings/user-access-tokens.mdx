作为组织所有者或管理员，您可以从单一集中页面查看和管理组织成员创建的所有个人访问令牌（PAT）。 这允许您监控 API 使用情况，并在必要时通过撤销令牌来增强安全性。

要管理组织的访问令牌，请转到 **组织设置 > 用户访问令牌**。

## 管理用户访问令牌

**用户访问令牌**页面显示组织内所有有效个人访问令牌的列表。

<Image src={userAccessTokens} alt="用户访问令牌页面" />

该表格为每个令牌提供以下信息：

* **名称**：创建令牌时为其指定的描述性名称。
* **令牌**：用于识别的部分掩码版本令牌。
* **创建者**：生成该令牌的组织成员。
* **创建时间**：令牌的创建日期。
* **最后使用时间**：令牌最后一次用于发出 API 请求的日期。
* **过期时间**：令牌的过期日期。 值为"从不"表示该令牌永不过期。

### 撤销用户访问令牌

您可以随时撤销任何成员的个人访问令牌。 如果令牌已泄露、与前员工相关联或不再需要，此操作非常有用。

要撤销令牌，请按照以下步骤操作：

<Steps>
  1. 转到 **组织设置 > 用户访问令牌**。
  2. 在列表中找到您要撤销的令牌，然后单击其旁边的 **撤销**。
</Steps>

<Aside type="caution" title="撤销立即生效">
  撤销令牌是一项永久性操作。 一旦撤销，该令牌将无法再用于访问 API，这可能会导致依赖该令牌的应用程序或脚本无法正常运行。
</Aside>
