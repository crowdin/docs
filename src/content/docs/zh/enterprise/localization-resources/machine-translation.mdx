与 Crowdin Enterprise 集成的机器翻译引擎提供来自 Google 翻译、AutoML 翻译、微软翻译等服务的翻译建议。

机器翻译建议按引擎配置日期的顺序显示在编辑器的翻译记忆和机器翻译建议部分。 如需优先显示特定引擎的建议，请先配置该引擎。

此外，您还可以使用机器翻译引擎执行[自动翻译](/zh/enterprise/auto-translation/)。

<Image src={machineTranslationSuggestions} alt="在编辑器中的机器翻译建议" />

<LinkCard title="添加更多机器翻译引擎" description="从 Crowdin Store 安装更多机器翻译引擎。" href="https://store.crowdin.com/tags/machine-translation" target="_blank" />

默认情况下，每个项目都启用机器翻译，但需要先配置翻译引擎，然后项目成员才能使用。 如果您想为特定项目禁用此选项，请清除[项目设置](/zh/enterprise/project-settings/machine-translation/)中的**显示机器翻译建议**。

### 配置机器翻译引擎

要配置机器翻译引擎，请遵循以下步骤：

<Steps>
  1. 打开您组织的**工作区**，然后选择左侧边栏上的**机器翻译**。
  2. 在右下角，单击 <Icon name="mdi:plus" class="inline-icon" />**添加机器翻译引擎**。
  3. 在出现的对话框中，从下拉列表中选择引擎。
  4. *（可选）* 如有需要，编辑机器翻译引擎的名称。
  5. 输入所需凭据。
  6. *（可选）* 选择适用的语言。 或者，留空以访问所有语言。
  7. *（可选）* 选择要使用该机器翻译引擎的项目。 或者，留空以对所有组织项目启用该引擎。
  8. 单击 **创建**。
</Steps>

### 微软翻译器

前往 [Windows Azure](https://portal.azure.com/#blade/HubsExtension/BrowseAll) 获取您的微软翻译 API 订阅密钥。 Translator Text API 可以免费提供 200 万个翻译字符。

<Aside>
  如果与您的微软翻译文本 API 密钥关联的*资源位置*不是*全局*，请确保在您的 Crowdin 账户中配置微软翻译机器翻译引擎时将其与其他凭据一起指定。
</Aside>

<LinkCard title="如何注册微软翻译文本 API" href="https://docs.microsoft.com/en-us/azure/cognitive-services/translator/translator-text-how-to-signup" target="_blank" />

### DeepL 翻译器

DeepL API 只能通过 DeepL API 免费和专业版订阅计划（[面向开发人员](https://www.deepl.com/pro.html#developer)标签）访问。 前往 [DeepL Pro 账户页面](https://www.deepl.com/pro-account.html)获取唯一的身份验证密钥。 请注意，DeepL API 计划是唯一提供此功能的计划。

如果您订阅了其他订阅计划，您可以在[个人账户设置](https://www.deepl.com/pro-account.html)中切换到 DeepL API 计划。 DeepL 还提供了一个[模拟器](https://www.deepl.com/docs-api/simulator)，允许您实际检查您的身份验证密钥。

#### DeepL 术语库配置

DeepL 翻译支持多语言术语库，让您能够确保特定领域术语的翻译一致性。 在 Crowdin Enterprise 中，您可以配置多个术语库并为每个术语库分配特定的语言对。

要配置您的 DeepL 术语库，请按照以下步骤操作：

<Steps>
  1. 从**术语库**下拉菜单中选择一个术语库。 此列表会自动从您连接的 DeepL 账户中填充。
  2. 从**语言对**下拉菜单中，选择所选术语库应适用的一个或多个语言对。
  3. *（可选）* 要配置另一个术语库，单击 <Icon name="mdi:plus-circle" class="inline-icon" /> 并重复上述步骤。 要删除术语库配置，单击 <Icon name="mdi:bin" class="inline-icon" />。
  4. 单击 **创建**。
</Steps>

<Image src={deeplGlossaryConfiguration} alt="DeepL 术语库配置" class="width-xl" />

<Aside type="tip" title="语言变体映射">
  Crowdin Enterprise 允许您映射不同的语言变体。 例如，如果您在 DeepL 中的术语库使用的是通用源语言（如“英语”），您可以将其映射到 Crowdin Enterprise 项目中的特定变体，例如“英语（美国）”。 这样可确保即使您项目的源语言是特定方言（例如，加拿大英语；墨西哥西班牙语），术语库也能正确应用。
</Aside>

### ModernMT 翻译

ModernMT 实时翻译 API 可通过实时或人机交互订阅计划访问。 前往 [ModernMT 仪表板](https://www.modernmt.com/dashboard/)获取您的许可证密钥。 如果您订阅了其他订阅计划，您可以通过[管理计划](https://www.modernmt.com/dashboard/manage-plan)页面切换到实时计划或人机交互计划。

### Amazon 翻译

Amazon 翻译的免费版本可使用 12 个月。 在此之后，您将需要为翻译的字符数付费。 Crowdin 使用 Amazon 翻译提供的标准文本翻译类型。 详细了解[可用的 Amazon 翻译定价选项](https://aws.amazon.com/translate/pricing/)。

详细了解如何[获取 Amazon 翻译集成所需的访问密钥](https://docs.aws.amazon.com/general/latest/gr/aws-sec-cred-types.html#access-keys-and-secret-access-keys)。

要使用您的[自定义术语](https://docs.aws.amazon.com/translate/latest/dg/how-custom-terminology.html)，请按照以下步骤操作：

<Steps>
  1. 指定您的 AWS 区域 (如 us-east-2)。
  2. 从相应的下拉菜单中选择您的自定义术语。
</Steps>

<QuestionAnswer title="在 Crowdin 中设置 Amazon 翻译所需的 IAM 用户最低权限范围是什么？">
  所需的最低 IAM 权限范围取决于您是否使用术语：

  **不使用术语：**

  ```json
  {
    "Version": "2012-10-17",
    "Statement": [
      {
        "Effect": "Allow",
        "Action": "translate:TranslateText",
        "Resource": "*"
      }
    ]
  }
  ```

  **使用术语：**

  ```json
  {
    "Version": "2012-10-17",
    "Statement": [
      {
        "Effect": "Allow",
        "Action": [
          "translate:TranslateText",
          "translate:GetTerminology",
          "translate:ListTerminologies"
        ],
        "Resource": "*"
      }
    ]
  }
  ```

  或者，您可以使用 [AWS 托管策略：TranslateReadOnly](https://docs.aws.amazon.com/translate/latest/dg/security-iam-awsmanpol.html#security-iam-awsmanpol-TranslateReadOnly)，该策略提供对 Amazon 翻译的必要只读访问权限。

  <Aside>`"Resource": "*"` 允许访问所有资源。 为了更严格的安全性，如果适用，请考虑将其限制为特定的 AWS 区域或资源。</Aside>
</QuestionAnswer>

### Google 翻译

要使用 Google 翻译，您应拥有一个具有有效结算资料的 Google Cloud 账户。 云翻译根据使用情况按月定价。 [了解更多](https://cloud.google.com/translate/pricing)。

要获取集成所需的 API 密钥，请按照以下步骤操作：

<Steps>
  1. 前往 [Google Cloud Console](https://console.cloud.google.com/)。 登录到现有的 Google Cloud 账户或设置一个新账户。
  2. 选择现有项目或使用**新建项目**添加项目。
  3. 前往 [API 和服务 > 库](https://console.cloud.google.com/apis/library)并搜索 *Cloud Translation API*。
  4. 单击**启用**。
  5. 前往 [API 和服务 > 凭据](https://console.cloud.google.com/apis/credentials/)并单击**创建凭据 > API 密钥**。
  6. 复制 API 密钥并单击**关闭**。
</Steps>

### Google Cloud AutoML 翻译

要使用 AutoML 翻译，您应拥有一个具有有效结算资料的 Google Cloud 账户。 Google AutoML 翻译每月根据您发送以供翻译的字符数定价。 [了解更多](https://cloud.google.com/translate/automl/pricing#translation_costs)。

要创建服务账户密钥并获取集成所需的凭据，请执行以下步骤：

<Steps>
  1. 前往 [Google Cloud Console](https://console.cloud.google.com/)。 登录到现有的 Google Cloud 账户或设置一个新账户。
  2. 选择现有项目或使用**新建项目**添加项目。
  3. 前往 [API 和服务 > 库](https://console.cloud.google.com/apis/library)并搜索 *Cloud AutoML API*。
  4. 单击**启用**。
  5. 下一步是创建服务账户私钥对。 服务账户由 Crowdin 使用。 应用程序使用服务账户进行授权 API 调用。
</Steps>

要设置身份验证并创建您的私钥，请执行以下步骤：

<Steps>
  1. 前往 [API 和服务 > 凭据](https://console.cloud.google.com/apis/credentials/)。
  2. 单击**创建凭据 > 服务账户**。
  3. 在*服务账户名称*字段中，输入一个描述此服务账户功能的名称。
  4. 单击**创建并继续**。
  5. 从*角色列表*中，选择*项目* > *Cloud Translation API 编辑器*，然后单击**继续**。
  6. 单击**完成**。
  7. 在 [API 和服务 > 凭据](https://console.cloud.google.com/apis/credentials/)中，单击已创建的服务账户。
  8. 切换到**密钥**选项卡。
  9. 单击**添加密钥 > 创建新密钥**。
  10. 选择 **JSON** 并点击**创建**。 包含密钥的 JSON 文件将下载到您的计算机。
</Steps>

#### Google Cloud AutoML 自定义术语库配置

Google Cloud AutoML 自定义术语库可让您以一致的方式翻译特定领域的术语。 您一次只能使用一个 Google Cloud AutoML 自定义模型。 如果您想在 Crowdin 企业版项目中使用多个自定义模型，请在您的 Crowdin 企业版账户下创建多个 Google Cloud AutoML 机器翻译引擎实例，并在每个机器翻译引擎的设置中指定不同的模型。 在通过机器翻译进行自动翻译期间，您可以交替使用 Google Cloud AutoML 和所需的模型。

要配置您的 Google Cloud AutoML 自定义术语库，请按照以下步骤操作：

<Steps>
  1. 打开您组织的**工作区**，然后选择左侧边栏上的**机器翻译**。
  2. 在右下角，单击 <Icon name="mdi:plus" class="inline-icon" />**添加机器翻译引擎**。
  3. 在出现的对话框中，从**引擎**下拉列表中选择 **Google AutoML Translate**。
  4. 单击**选择文件**并上传您的 JSON 凭据。
  5. *（可选）* 指定您的自定义模型。 <Aside>如果在机器翻译引擎设置中未指定 Google Cloud AutoML 自定义模型，则将使用 Google NMT 模型代替。</Aside>
  6. 指定您的资源位置（例如 us-central1）。
  7. 从相应的下拉列表中选择您的术语库。
  8. *（可选）* 配置自定义模型与 Crowdin 语言之间的语言映射。
  9. *（可选）* 选择适用的语言。 或者，留空以访问所有语言。
  10. *（可选）* 选择要使用该机器翻译引擎的项目。 或者，留空以对所有组织项目启用该引擎。
  11. 单击 **创建**。
</Steps>

## 机器翻译引擎 ID

指定机器翻译引擎凭据后，系统会自动分配一个唯一的机器翻译引擎 ID。 稍后，您可以在 API 请求中使用这些 ID 通过机器引擎进行自动翻译。

您可以在组织的**工作区 > 机器翻译**页面中找到机器翻译引擎 ID。
或者，您也可以[通过 API 获取机器翻译引擎 ID](/developer/enterprise/api/v2/#tag/Machine-Translation-Engines/operation/api.mts.getMany)。

## 清除机器翻译引擎缓存

当机器翻译引擎更新了新功能或模型时，Crowdin Enterprise 中的缓存翻译可能不再反映最新的翻译能力。 要在编辑器的**翻译记忆和机器翻译建议**部分显示更新后的翻译，请清除机器翻译引擎缓存。 如果您已在编辑器中查看过某些字符串，这一功能尤为有用，因为 Crowdin Enterprise 会为这些字符串存储缓存的机器翻译建议。

要清除已配置的机器翻译引擎的缓存，请按照以下步骤操作：

<Steps>
  1. 打开您组织的**工作区**，然后选择左侧边栏上的**机器翻译**。
  2. 在列表中的所需机器翻译引擎上单击 <Icon name="mdi:dots-vertical" class="inline-icon" />（或右键单击）。
  3. 单击**清除缓存**。
</Steps>
