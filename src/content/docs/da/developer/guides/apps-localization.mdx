Håndtér effektivt lokalisering til både Android- og iOS-apps uden at oversætte de samme strenge to gange.

## Lokaliseringshåndtering inden for ét Enkelt Projekt

Oversæt Android- og iOS-filer i ét Crowdin-projekt. Brug en af mulighederne foreslået nedenfor afhængigt af de aktuelle projektspecifikationer.

Anbefalinger:

* Placér Android (Android XML) og iOS (iOS-strenge, iOS XLIFF) app-filer på forskellige steder (mapper) i Crowdin-projektet.
* Tilslut både Android- og iOS-repo'er til det relevante Crowdin-projekt for at automatisere synkroniseringen af kilde- og oversættelsesfilerne.
* Use the **Duplicate Strings** option to manage duplicates.
* Use the **Unify Placeholders** option to manage Android and iOS placeholders.

### Skjul dubletter

Upload iOS and Android app files to one Crowdin project and select the **Hide** option in the project's **Settings > Import > Source Strings**.

Normalt deler apps udviklet til Android- og iOS-platforme de fleste af kildestrengene til Android- og iOS-filer. So once you select the **Hide** option, the system will detect the duplicate strings for both types of files (iOS and Android) and hide them while keeping visible only the strings in the files uploaded first.

Once the string that was uploaded first is translated, the hidden duplicate will get this translation automatically due to the selected **Hide** option. På denne måde vil oversættere kun behandle de unikke, synlige tekster. Når oversættelsen af den synlige streng opdateres, opdateres oversættelserne for dens skjulte dubletter også.

  Read more about [Duplicate Strings](/project-settings/import/#duplicate-strings).

### Parameterunificering

If some source strings for iOS and Android are the same but differ only in placeholders, we recommend selecting the **Unify Placeholders** option in the project's **Settings > Import > Source Strings**.

For example, you added the iOS string `Hello, %@!` and a similar one to Android `Hello, %s!`. The **Unify Placeholders** option will convert both to `Hello, [%s]!`, so translation from the Android file can migrate to iOS. Ved eksport fås oversættelser med de oprindelige variabler/pladsholdere.

It’s also possible to add/modify new strings online in the project for Android XML and iOS Strings files via the **Strings** section of the project.

## Lokaliseringshåndtering inden for ét enkelt projekt med pakker

Anbefalinger:

* Upload enten iOS (iOS Strings, iOS XLIFF) eller Android (Android XML) app-filer til Crowdin-projektet.
* Opsæt paker for at kunne eksportere oversættelser i det nødvendige filformat.

<ReadMore>
  Read more about [Bundles](/bundles/).
</ReadMore>

### Oversættelseseksport via pakker

Lokalisér ressourcer til blot én enkelt app i Crowdin og download forskellige filformater til både Android- og iOS-apps.

For example, you can upload an XML file to Crowdin for [Android localization](https://crowdin.com/blog/2022/08/10/android-app-localization-tutorial) and receive two files on export: XML for Android and Strings for iOS. Der skal muligvis foretages nogle småjusteringer af de eksporterede filer (oversættelsesnøgler forbliver de samme som i Android-filen, så de måske har brug for justeringer for iOS-filen). Lokaliseringstiden og udgifterne til oversættelsestjenester kan dog reduceres betydeligt vha. denne tilgang.

## Næste trin i Mobil-apps lokalisering

Her er de næste trin, som kan overvejes under mobil-app lokaliseringen. Som et alternativ til en mere traditionel tilgang til kildefilhåndtering, kan strenge sendes til oversættelse direkte fra egne designværktøjer via Crowdin-plugins. Another good option is to use CDN Distributions to update translated strings of your mobile apps instantly without a need to roll out a new version on the App Store or Google Play.
