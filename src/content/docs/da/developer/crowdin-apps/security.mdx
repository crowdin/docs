To ensure the high level of security for cases when the Crowdin app works with the data from Crowdin (i.e. uses the authorization via `crowdin_app`), we've developed a security mechanism. The main principle of this security mechanism is based on the exchange of the JWT token between Crowdin and the Crowdin app. JWT-tokenet er signeret med en OAuth Client-hemmelighed kun kendt af de to deltagende parter. This way, the Crowdin app can get a confirmation that the page is opened precisely in Crowdin.

## Implementering

Følg disse trin for at implementere godkendelse og bekræftelse i Crowdin-appen:

* Add the authorization via `crowdin_app` to your app descriptor and add the OAuth Client ID that will be used for authorization.
* Add the callback to your Crowdin app that will handle the [Installed Event](/developer/crowdin-apps-installation/#installed-event-communication-flow).
* Angiv det relevante sæt af områder krævet for Crowdin-appen i app-deskriptor'en. Det angivne sæt af områder må ikke overskride områderne angivet i OAuth.

Using the above methods, on each request to the Crowdin app, Crowdin will pass a set of parameters along with a security token, which can be validated by a secret from the OAuth.

Below you can see an example of the URL used by Crowdin to open a module page.

```shell wrap
https://example.com/app-module?jwtToken=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJhdWQiOiJCcjRhMmhwUW……MX0.yt-lbv3Z8JyIGX4jG405mjZvX8lwc1q0EfWdTtm9GCc&origin=https://{domain}.crowdin.com&clientId=your-client-id
```

Bedste praksis vil være at føje middleware til Crowdin-appen for at bekræfte, hvorvidt hver forespørgsel har et token med en gyldig signatur og udløbsdato. You can use one of the [existing libraries](https://jwt.io/) to validate the authenticity of the token.

## JWT-tokenstruktur

Et JWT-token består af flg. dele:

* Header &ndash; indeholder information om typen af token og kodningsalgoritmer.
* Payload &ndash; indeholder yderligere oplysninger om udstedelses- og udløbsdatoer for tokenet, oplysninger om tokenudstederen og anmoderen samt yderligere kontekstuel information.
* Signatur &ndash; delen med en signatur baseret på header og payload.

JWT-token payload eksempel:

```json
{
  "aud": "Br4a2hpQiNW96anuuO4a",
  "sub": "1",
  "domain": null,
  "context": {},
  "iat": 1600000000,
  "exp": 1600000900
}
```
