Crowdin, kullanıcıların farklı ihtiyaçlarını karşılamak için aşağıdakiler gibi çeşitli bildirim türleri sunar:

* *Project Updates*: Notify users of changes to the project, such as new content to translate or updates to existing content.
* *Translation Activity*: Notify users of new translations, approvals, or comments on translations.
* *Task Management*: These notifications inform users about new tasks, deadlines and task completions.
* *Team Communication*: These alerts notify users of messages and discussions within the project.

Bildirimleri görmek için sayfanın sağ üst köşesindeki <Icon name="mdi:bell-outline" class="inline-icon" /> zil simgesine tıklayın.

<LinkCard title="Notification Settings" href="/tr/account-settings/#bildirimler" />

## Slack Bütünleştirmesi

Doğrudan Slack’e gönderilen güncellemelerle, yönettiğiniz veya katkıda bulunduğunuz Crowdin projelerinde neler olduğunu anında bileceksiniz. Crowdin hesabınızı Slack ile bütünleştirmenizden sonra, almak istediğiniz bildirimleri seçin, Crowdin botu bunları doğrudan ileti olarak gönderecektir.

<Aside>
  Slack’ta bildirim almak isteyen her bir kullanıcı için Crowdin için Slack bütünleştirmesi yapılandırılmak zorundadır.
</Aside>

Slack aracılığıyla Crowdin bildirimlerini almak için şu adımları izleyin:

<Steps>
  1. Sağ üst köşedeki profil resminize tıklayın ve **Ayarlar**’ı seçin.
  2. **Bildirimler** sekmesine geçin ve **Slack’ı Bağla**’ya tıklayın. <Image src={slackConnect} alt="Slack’ı Bağla" />
  3. Slack tarafında Crowdin ile bağlantıyı yetkilendirin. <Image src={slackAuthorize} alt="Authorize Slack" class="width-lg" />
  4. Crowdin’e geri dönün ve Slack’te almak istediğiniz bildirimleri seçin.
</Steps>

<ReadMore>
  [Genel Bildirim Ayarları](/tr/account-settings/#bildirimler) hakkında daha fazlasını okuyun.
</ReadMore>

Crowdin bot’tan doğrudan ileti olarak seçilen bildirim türlerini alacaksınız.

<Image src={slackCrowdinNotifications} alt="Crowdin Notifications in Slack" />

İstediğiniz zaman bildirimleri etkisizleştirebilir ve Slack’in Crowdin’den bağlantısını kesebilirsiniz.

## Özel Bildirimler

Özel Bildirimler, Crowdin’de meydana gelen yeni olaylar hakkında bilgi almanızı sağlar. Hesabınız için özel bildirimleri yapılandırdıktan sonra Crowdin, HTTP aracılığıyla özel bildirim URL’sine veriler içeren POST istekleri göndermeye başlayacaktır.

### Kullanım Durumları

Hizmetlerle veya arka ucunuzla bütünleştirmeler oluşturmak için özel bildirimleri yapılandırabilirsiniz. Örneğin:

* Kullandığınız sisteme bildirim göndermek için özel bildirimleri ayarlayın.
* Bilgileri belirli istek gereksinimleriyle (örn. içerik türü, başlıklar, yük) üçüncü taraf hizmetlerine aktarın.
* Crowdin ile özel bütünleştirmeler oluşturun.

### Configuring

Özel bildirimleri Crowdin’de yapılandırmak için şu adımları izleyin:

<Steps>
  1. **Hesap Ayarları**’nızı açın ve **Bildirimler** sekmesine gidin.
  2. **Bildirimler Özel Kanalı** bölümünde, özel bildirimlerinizi yapılandırmaya başlamak için **Bildirimler Özel Kanalını Ayarla**’ya tıklayın.
</Steps>

Özel bildirimleri ayarlamak için aşağıdaki bilgileri sağlamanız gerekecek:

* Geri çağrının gönderilmesi gereken URL.
* The content type for the POST request method (`multipart/form-data`, `application/json`, or `application/x-www-form-urlencoded`).

İsteğe bağlı olarak, özel bildirimlerinize özel başlıklar ekleyebilirsiniz. Yetkilendirme yöntemi ve daha fazlası olarak ek güvenlik için kullanılabilirler. Örneğin, eğer başlık eklerseniz, özel bildirim uç noktanız bunları doğrulayabilir ve bu bilgilerin Crowdin’den geldiğinden emin olabilir.

When configuring custom notifications, click **Test notification** to see how your application will react to that call. Once finished with the configuration, click **Save**.

<Image src={customNotificationsSetUp} alt="Özel Bildirimleri Ayarlama" class="width-xl" />

Özel bildirim yönetimine yaklaşımınıza bağlı olarak, Crowdin’in önceden yapılandırılmış özel bildirimler URL’lerini açmasına izin vermek için güvenlik duvarınıza adanmış Crowdin IP adreslerini eklemeniz gerekebilir.

<ReadMore>
  Read more about [IP Addresses](/developer/ip-addresses/).
</ReadMore>

### Özel Yükler

Öğeleri sisteminizin gerektirdiği şekilde eklemek ve düzenlemek için özel bildirim yükünü değiştirebilirsiniz. To get informative and user-friendly notifications, make sure to include the `{{notification-message}}` placeholder in your payload.

### Configuring for Discord {#discord}

Discord’da Crowdin’den gelen Özel Bildirimler ayarlamak için aşağıdaki adımları izleyin.

#### Discord Hesabı Oluşturma

If you don't have a Discord account already, you will need to [create one](https://discord.com/register).

<Aside>
  Crowdin’den gelen özel bildirimleri sadece Discord masaüstü uygulaması veya Discord web arayüzü aracılığıyla yapılandırabilirsiniz. Yapılandırıldıktan sonra, bu bildirimleri Discord mobil uygulamaları aracılığıyla da alabileceksiniz.
</Aside>

Download and install the [Discord desktop app](https://discord.com/download). Alternatif olarak, daha fazla yapılandırma için Discord web arayüzünü kullanabilirsiniz.

#### Discord Web Kancası Oluşturma

Discord’da bildirim almak için bir web kancası oluşturmanız gerekir. Tamamlandıktan sonra web kancası URL’sini kopyalayın. Crowdin tarafındaki yapılandırmalar için buna ihtiyacınız olacak.

<ReadMore>
  Read more about [creating a Discord webhook](https://support.discord.com/hc/en-us/articles/228383668-Intro-to-Webhooks).
</ReadMore>

#### Configuring Custom Notifications Channel {#custom-channel-discord}

<Steps>
  1. **Hesap Ayarları > Bildirimler**’inizi açın.
  2. **Bildirimler Özel Kanalını Ayarla**’ya tıklayın.
  3. In the appeared dialog, paste the Discord webhook URL in the **URL** field.
  4. Select **application/json** for the **Content type**.
  5. Paste the following payload in the **Payload** field:
     ```json
     {
       "content": "{{notification-message}}"
     }
     ```
  6. Crowdin’den Discord kanalınıza bir deneme iletisi almak için **Bildirimi dene**’ye tıklayın.
  7. Once finished with the configuration, click **Save**.
</Steps>

### Configuring for Telegram {#telegram}

Telegram’da Crowdin’den gelen Özel Bildirimler ayarlamak için aşağıdaki adımları izleyin.

#### Telegram Hesabı Oluşturma

Eğer zaten bir Telegram hesabınız yoksa, bir telefonla bağlanmanız gerekecektir. Telegram, telefon numaranızı, hesabınıza oturum açmak için kullanacağınız birincil kimlik bilgisi olarak kullanır.

So firstly, download and install the Telegram app using [Android](https://telegram.org/dl/android) or [iOS](https://telegram.org/dl/ios).
Once signed up, you may use the [Telegram web interface](https://telegram.org/dl/webogram) with a PC or Mac to simplify further configurations, but this part is up to you.

#### Telegram Bot’u Oluşturma

Telegram’da bildirim almak için bir bot oluşturmanız gerekir. Bitirdikten sonra, size bir bot’un API belirteci verilecektir. Kaydettiğinizden emin olun; daha sonra ihtiyacınız olacak.

<ReadMore>
  Read more about [creating a Telegram bot](https://core.telegram.org/bots#creating-a-new-bot).
</ReadMore>

#### Telegram Kanalını Yapılandırma

Bir sonraki adım Crowdin’den bildirim almak için kullanılacak bir genel Telegram kanalı oluşturmaktır. Later you'll be able to change the channel's privacy if needed.

Bir Telegram kanalı oluşturmak için şu adımları izleyin:

<Steps>
  1. Cihazınızda Telegram’ı açın.
  2. Click on the pen icon and select **New Channel**.
  3. Name your channel in the **Channel name** field.
  4. *(Optional)* Specify a channel description.
  5. *(Optional)* Click on the camera icon to set a display picture for your channel.
  6. Click **Next** in the top-right corner.
  7. Select **Public** channel type.
  8. Kanalınız için kalıcı bir bağlantı belirtin. Bu bağlantı, insanların kanalınızı aramak ve katılmak için kullanacağı bağlantıdır.
  9. Click **Next** in the top-right corner.
  10. *(Optional)* In this step, Telegram will ask you to add subscribers to your Telegram channel. Yeni bot’unuz da dahil olmak üzere eklemek istediğiniz kişileri seçin (bot’unuza Üst Yönetici izinleri verin). Daha sonra ekleyebileceğinizden şimdilik herhangi bir üye eklememeyi seçebilirsiniz.
  11. Click **Next** in the top-right corner to continue and create your channel on Telegram.
</Steps>

#### Configuring Custom Notifications Channel {#custom-channel-telegram}

<Steps>
  1. **Hesap Ayarları**’nızı açın ve **Bildirimler** sekmesine gidin.
  2. **Bildirimler Özel Kanalını Ayarla**’ya tıklayın.
  3. In the appeared dialog, specify the **URL** in the following format: `https://api.telegram.org/bot{bot_API_token}/sendMessage`.
  4. Select **application/json** for the **Content type**.
  5. Paste the following payload in the **Payload** field:
     ```json
     {
       "chat_id": "{public_chat_id}",
       "text": "{{notification-message}}"
     }
     ```
     where `{public_chat_id}` &ndash; Your public chat id you specified for your channel link during the Telegram channel configuration.
  6. Crowdin’den Telegram kanalınıza bir deneme iletisi almak için **Bildirimi dene**’ye tıklayın.
  7. Once finished with the configuration, click **Save**.
</Steps>

#### Telegram Kanal Türünü Değiştirme

Eğer Telegram kanalınızı özel yapmak ve Crowdin’den bildirimleri almaya devam etmek istiyorsanız, şu adımları izleyin:

<Steps>
  1. First of all you need to acquire your Telegram channel's original chat id by visiting: `https://api.telegram.org/bot{bot_API_token}/sendMessage?chat_id={public_chat_id}&text=Test`
  2. Telegram kanalınızın orijinal sohbet kimliğini içeren JSON biçiminde bir yanıt alacaksınız.
  3. Crowdin’de **Hesap Ayarları**’nızı açın ve **Bildirimler** sekmesine gidin.
  4. **Bildirimler Özel Kanalını Ayarla**’ya tıklayın.
  5. In the appeared dialog, modify the **URL** by replacing the chat id with the one received in the JSON response above.
  6. **Kaydet**’e tıklayın.
  7. Telegram’da kanalınızı açın.
  8. Click on your channel name and click **Edit**.
  9. Click **Channel Type**.
  10. Select **Private** and click **Done**.
</Steps>

<Aside>
  Make sure to replace `{bot_API_token}` and `{public_chat_id}` with your actual Telegram bot API token and public chat id.
</Aside>
